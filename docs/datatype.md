# C% 데이터 타입

## 논리형

| 타입   | 크기  | 값              | 비고 |
| ------ | ----- | --------------- | ---- |
| `bool` | 1비트 | `true`, `false` |      |

- 리터럴: `true`, `false`
- 비교 연산(`==`, `!=`, `<`, `>`, `<=`, `>=`)의 결과 타입
- 논리 연산: `&&` (AND), `||` (OR), `!` (NOT)
- 숫자 변환: `true` → 1, `false` → 0
- 기본값: `false`

## 정수형

### 고정 크기 정수

| 타입      | 크기   | 최솟값                     | 최댓값                    | 오버플로    |
| --------- | ------ | -------------------------- | ------------------------- | ----------- |
| `int`   | 플랫폼 의존 | `int16f`의 별칭 (기본: 64비트) | — | wrap-around |
| `int8`  | 8비트  | -128                       | 127                       | wrap-around |
| `int16` | 16비트 | -32,768                    | 32,767                    | wrap-around |
| `int32` | 32비트 | -2,147,483,648             | 2,147,483,647             | wrap-around |
| `int64` | 64비트 | -9,223,372,036,854,775,808 | 9,223,372,036,854,775,807 | wrap-around |

### 부호 없는 정수 (Unsigned)

| 타입      | 크기   | 최솟값 | 최댓값                     | 오버플로    |
| --------- | ------ | ------ | -------------------------- | ----------- |
| `uint`    | 플랫폼 의존 | 0 | `uint16f`의 별칭 (기본: 64비트) | wrap-around |
| `uint8`   | 8비트  | 0      | 255                        | wrap-around |
| `uint16`  | 16비트 | 0      | 65,535                     | wrap-around |
| `uint32`  | 32비트 | 0      | 4,294,967,295              | wrap-around |
| `uint64`  | 64비트 | 0      | 18,446,744,073,709,551,615 | wrap-around |

- `uint`는 `uint16f`의 별칭 (플랫폼에 따라 크기 변동)
- 음수 대입 시 wrap-around: `uint8 x = -1;` → `x == 255`
- `uint64.max()`는 BigInt로 반환 (int64 범위 초과)

### Fast 정수 (플랫폼 최적 크기)

최소 비트 수를 보장하되, 플랫폼에서 가장 빠른 크기로 결정되는 타입. C++의 `int_fastN_t`에 대응한다.

| 타입       | 최소 크기 | C++ 매핑           | 비고 |
| ---------- | --------- | ------------------ | ---- |
| `int8f`    | 8비트     | `int_fast8_t`      |      |
| `int16f`   | 16비트    | `int_fast16_t`     |      |
| `int32f`   | 32비트    | `int_fast32_t`     |      |
| `uint8f`   | 8비트     | `uint_fast8_t`     |      |
| `uint16f`  | 16비트    | `uint_fast16_t`    |      |
| `uint32f`  | 32비트    | `uint_fast32_t`    |      |

- `int`는 `int16f`의 별칭 — 플랫폼에 따라 크기가 달라진다
- `uint`는 `uint16f`의 별칭
- 실제 크기는 `--platform` 옵션으로 결정 (기본값: 64비트)
- **오버플로: 런타임 에러** (고정 크기 타입의 wrap-around와 다름)
- `int64f`/`uint64f`는 없음 — 모든 플랫폼에서 64비트 고정이므로 `int64`/`uint64`와 동일

#### 플랫폼별 fast 타입 크기 (비트)

| 플랫폼     | fast8 | fast16 | fast32 |
| ---------- | ----- | ------ | ------ |
| `default`  | 8     | 64     | 64     |
| `x86`      | 8     | 32     | 32     |
| `x64-linux`| 8     | 64     | 64     |
| `x64-win`  | 8     | 32     | 32     |
| `avr`      | 8     | 16     | 32     |
| `arm32`    | 32    | 32     | 32     |
| `arm64`    | 8     | 64     | 64     |
| `esp32`    | 8     | 32     | 32     |

### 동적 정수

| 타입       | 크기 | 오버플로    | 비고                          |
| ---------- | ---- | ----------- | ----------------------------- |
| `intbig` | 가변 | BigInt 승격 | int64 범위에서는 고속, 오버플로 시 임의 정밀도 |
| `bigint` | 가변 | 없음        | 항상 임의 정밀도              |

- 모든 정수형은 **숫자**로만 취급된다. `int8`은 C/C++의 `char`와 달리 문자가 아니다.
- `int`: `int16f`의 별칭. 플랫폼에 따라 크기가 달라진다 (기본 64비트). 오버플로 시 런타임 에러
- `int8`~`int64`: 오버플로 시 wrap-around
- `intbig`: int64 범위에서는 fast path, 오버플로 시 자동으로 BigInt 승격, int64 범위로 돌아오면 자동 강등
- `bigint`: 항상 BigInt (임의 정밀도). 승격/강등 없음
- 기본값: `0`

> **NOTE**: 현재 cpct-interpreter는 프로토타입 단계로, 모든 고정 크기 정수(`int`, `int8`~`int64`, `uint`~`uint64`, `char`)는 내부적으로 `int64`로 저장된다. 범위 제한은 대입/연산 시 소프트웨어적으로 적용된다. 자세한 내용은 [docs_dev/internals.md](../docs_dev/internals.md) 참고.

## 실수형

| 타입      | 크기   | 정밀도        | 범위 (절댓값)              | 비고            |
| --------- | ------ | ------------- | -------------------------- | --------------- |
| `float`   | 64비트 | ~15-17 자릿수 | ~5e-324 ~ ~1.8e+308        | `float64` alias |
| `float32` | 32비트 | ~6-9 자릿수   | ~1.2e-38 ~ ~3.4e+38        | 단정밀도        |
| `float64` | 64비트 | ~15-17 자릿수 | ~5e-324 ~ ~1.8e+308        | 배정밀도        |

- `float`는 `float64`의 alias (C의 `double`에 해당)
- `float32`는 단정밀도 (C의 `float`에 해당), 대입/연산 시 32비트 정밀도로 절삭
- `++`/`--` 연산 지원 (C/C++과 동일하게 1.0을 더하거나 뺌)
- 기본값: `0.0`

## 문자형

| 타입   | 크기  | 최솟값 | 최댓값 | 오버플로    | 비고                          |
| ------ | ----- | ------ | ------ | ----------- | ----------------------------- |
| `char` | 8비트 | -128   | 127    | wrap-around | 내부적으로 정수, 출력 시 문자 |

- `char`는 내부적으로 `int8`과 동일한 범위의 정수로 저장
- 리터럴: `'A'`, `'\n'`, `'\t'`, `'\\'`, `'\''`, `'\0'`
- `print()` 시 문자로 출력 (정수가 아닌 해당 문자)
- 산술 연산 가능 (정수처럼 취급)
- `++`/`--` 연산 지원
- 기본값: `0` (널 문자)

## 문자열

| 타입     | 크기 | 비고           |
| -------- | ---- | -------------- |
| `string` | 동적 | UTF-8 문자열   |

- 리터럴: `"hello"`, `"hello\nworld"`
- 문자열 리터럴은 내부적으로 `char[]`로 저장 (성능), `string` 변수에 대입 시 `string`으로 승격
- `+` 연산은 항상 **문자열 연결**: `"hello" + 2` → `"hello2"` (자동 변환)
- Raw 문자열: `r"C:\path\to\file"` — 이스케이프 시퀀스 무시
- F-string: `f"Hello, {name}!"` — `{}` 안에 표현식 삽입
- `len()` 함수로 길이 조회
- 기본값: `""` (빈 문자열)

> **NOTE**: 문자열 리터럴(`"..."`)의 `+`는 항상 문자열 연결이다. `char` 변수의 `+`는 산술 연산이다. 포인터가 없으므로 C/C++의 포인터 산술 모호성이 존재하지 않는다.
